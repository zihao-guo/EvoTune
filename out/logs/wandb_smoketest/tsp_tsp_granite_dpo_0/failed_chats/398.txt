```python
import numpy as np
from scipy.sparse.linalg import eigsh

def heuristics(distance_matrix):
    # Calculate degree centrality
    degree_centrality = np.array(np.sum(distance_matrix, axis=0))
    
    # Calculate betweenness centrality
    # Betweenness Centrality is a measure of the number of shortest paths that pass through a particular vertex
    # We will calculate it for each vertex (row)
    n = distance_matrix.shape[0]
    betweenness = np.zeros((n, n))
    for _ in range(n):
        # Initialize adjacency matrix with ones for undirected graph
        adj_matrix = np.eye(n)
        for i in range(n):
            for j in range(n):
                if distance_matrix[i, j]!= 0:  # Avoid zero division
                    adj_matrix[i, j] = 1 / distance_matrix[i, j]
        
        # Betweenness calculation
        short_paths = eigsh(adj_matrix - np.eye(n), k=1)[0]
        betweenness += short_paths * (short_paths > 0)
    
    # Normalize centrality measures
    degree_centrality = degree_centrality / np.sum(degree_centrality)
    betweenness = betweenness / np.sum(betweenness)
    
    # Combine with distance matrix
    heuristics_matrix = degree_centrality + betweenness + np.abs(distance_matrix)
    
    return heuristics_matrix
```

This heuristics function introduces two new indicators:

1. Degree Centrality: The number of edges connected to each vertex. It's a simple measure of how important each node is in the network.

2. Betweenness Centrality: A measure of the number of shortest paths that pass through a particular vertex. It indicates how much a node 'bridges' or 'controls' information flow in the network.

These two measures, combined with the absolute distance values, provide a more nuanced assessment of each edge's importance, potentially outperforming the original function. However, actual performance can vary depending on the specific characteristics of the input distance matrices.
