This could be achieved by introducing a 'centrality measure' such as degree centrality, betweenness centrality, or eigenvector centrality, which considers the connectivity and influence of each node.

Let's consider using Degree Centrality for this task. This measure assigns higher weights to nodes (edges) that connect to more other nodes. This would imply that edges connecting to frequently connected nodes are more desirable than isolated edges.

Here's a Python function using NetworkX library for computing degree centrality:

```python
import numpy as np
import networkx as nx

def heuristics(distance_matrix):
    # Convert distance matrix to graph
    G = nx.from_numpy_array(distance_matrix)
    
    # Calculate degree centrality
    degree_centrality = nx.degree_centrality(G, weight='weight')  # assuming the distance matrix represents weights
    
    # Normalize degree centrality values
    max_centrality = max(degree_centrality.values())
    normalized_centrality = {node: centrality / max_centrality for node, centrality in degree_centrality.items()}
    
    # Return the normalized degree centrality as a matrix
    return np.array(list(normalized_centrality.values()))
```
