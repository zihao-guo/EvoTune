```python
import numpy as np

def kruskal_heuristics(distance_matrix):
    n = distance_matrix.shape[0]
    # Sort all edges in non-decreasing order
    sorted_edges = np.argsort(distance_matrix, axis=1)
    
    # Initialize a new matrix to store the heuristic scores
    heuristics_matrix = np.zeros_like(distance_matrix)
    
    # Iterate over sorted edges
    for i in range(n):
        for j in range(i+1, n):
            # If the edge is not already included, add its heuristic score
            if sorted_edges[i][j]!= -1:
                heuristics_matrix[i, j] = heuristics_matrix[j, i] = distance_matrix[i, j]
            else:
                # If not included, include it with its heuristic score
                heuristics_matrix[i, j] = heuristics_matrix[j, i] = distance_matrix[i, j] + (i + j) * 2  # (i + j) * 2 is an arbitrary growth factor
                break
    
    return heuristics_matrix
```

This function creates a weighted MST by considering the order of edges. A lower heuristic score means an edge should be included, while a higher score implies it shouldn't. The factor (i + j) * 2 is added to incentivize including edges that belong to larger, potentially more beneficial paths.

The score is determined by this formula:

`heuristic_score = edge_weight + (i + j) * 2`

where `i` and `j` are the indices of the two vertices connected by the edge.

Please note that this is a simple heuristic and might not always yield the optimal solution for complex problems. For more accurate results, especially for larger and more complex graphs, more advanced heuristics or exact algorithms such as dynamic programming could be employed.
