```python
import numpy as np

def weighted_heuristics(distance_matrix):
    num_nodes = distance_matrix.shape[0]

    # Calculate density (number of edges) per node
    node_density = np.sum(distance_matrix, axis=1) / num_nodes

    # Calculate centrality (betweenness centrality for simplicity)
    betweenness_centrality = np.zeros(num_nodes)
    for _ in range(10):  # Run the calculation 10 times to improve accuracy
        betweenness = np.zeros(num_nodes)
        for i in range(num_nodes):
            for j in range(num_nodes):
                if i!= j:
                    sub_matrix = distance_matrix[np.ix_(np.concatenate([np.arange(num_nodes)!= i, np.arange(num_nodes)!= j]), np.concatenate([np.arange(num_nodes)!= i, np.arange(num_nodes)!= j])))
                    betweenness[i] += sub_matrix[j, :].sum() / (sub_matrix.shape[0] - 1)
        betweenness_centrality = (betweenness / np.sum(betweenness)) if np.sum(betweenness) > 0 else betweenness

    # Combine distance and density/centrality metrics
    heuristic_matrix = -distance_matrix - node_density / 2 - betweenness_centrality  # Negative to maximize
    return heuristic_matrix
```

This new function considers:

1. Edge density: The number of edges connected to each node.
2. Node centrality: Betweenness centrality, which measures the influence of a node in the network.

These factors combine to give a more comprehensive heuristic score for each edge. The negative sign ensures that the resultant matrix maximizes the score, which in turn should lead to a better overall solution.

The density and centrality metrics are normalized by dividing by the number of nodes and the maximum centrality value, respectively, to ensure their impacts scale with the size of the graph.
