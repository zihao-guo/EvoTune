To improve the performance of the `heuristics()` function, let's introduce an approach based on the concept of "Cost-Complexity Principle" in graph theory. The idea is to balance the sum of the edge weights (costs) and a complexity term that encourages solutions with fewer edges.

We'll define two heuristics:
1. A simple heuristic that minimizes the total edge cost.
2. An enhanced heuristic that also considers the number of edges as a negative penalty.

```python
import numpy as np

def simple_heuristics(distance_matrix):
    # Simple heuristic: Minimize the sum of distances
    return distance_matrix.sum(axis=0)

def enhanced_heuristics(distance_matrix):
    # Enhanced heuristic: Minimize the sum of distances, but penalize it with a complexity term
    num_nodes = distance_matrix.shape[0]
    complexity_penalty = num_nodes * 10  # Adjust this value for desired trade-off

    # Total heuristic score
    heuristics_score = distance_matrix.sum(axis=0) + complexity_penalty

    return heuristics_score

# Now let's test both
distance_matrix = np.array([[0, 10, 15, 20],
                           [10, 0, 35, 25],
                           [15, 35, 0, 30],
                           [20, 25, 30, 0]])

print("Simple Heuristics Score: ", simple_heuristics(distance_matrix).sum())
print("Enhanced Heuristics Score: ", enhanced_heuristics(distance_matrix).sum())
```

This enhanced heuristics function introduces a 'complexity' penalty, which is calculated as a fraction of the total number of nodes. By doing so, it encourages solutions with fewer edges, which is a reasonable assumption in many real-world scenarios. The complexity penalty value (currently set to 10 in the `complexity_penalty` variable) can be adjusted to fine-tune the balance between minimizing total cost and minimizing the number of edges.
