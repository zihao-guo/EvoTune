We're applying a simple yet effective strategy here: high-quality edges (those closer together) will have higher desirability scores, while low-quality edges will have lower ones.

This approach takes into account the relative distances rather than just the absolute values. This could be more beneficial for certain types of problems where the quality of an edge (the connection between two nodes) is more important than its absolute distance.

Here's the Python function implementing this idea:

```python
def heuristics(distance_matrix):
    num_nodes = distance_matrix.shape[0]
    desirability_scores = [[1/(1+dist) for dist in distance_matrix[i]] for i in range(num_nodes)]
    return desirability_scores
```

In this function, each entry `desirability_scores[i][j]` is calculated as 1 divided by 1 plus the distance `distance_matrix[i][j]` between nodes `i` and `j`. This creates a desirability score that increases as the distance decreases, achieving the desired effect of prioritizing higher-quality edges.

The function returns a desirability matrix of the same shape as the input distance matrix, with each entry now representing the relative 'desirability' of the corresponding edge.
