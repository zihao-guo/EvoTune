Here's a simple yet novel approach using the concept of "negative impact" on a graph's global fitness.

This function will consider edges as having an "impact" score, calculated as the sum of the distance of its endpoints minus a scaling factor. Edges that connect vertices far apart will be considered more undesirable (higher impact).

```python
def heuristics(distance_matrix):
    n = len(distance_matrix)
    scaling_factor = 0.05  # A small scaling factor to give more weight to larger distances

    # Initialize a new matrix to store the impact scores
    impact_matrix = [[0 for _ in range(n)] for _ in range(n)]

    # Calculate the impact for each edge
    for i in range(n):
        for j in range(n):
            impact_matrix[i][j] = distance_matrix[i][j] - scaling_factor*(i + j)  # Adjust scaling factor as needed

    return impact_matrix

# Example Usage:
distance_matrix = [[0, 5, 10, 15],
                  [5, 0, 3, 8],
                  [10, 3, 0, 12],
                  [15, 8, 12, 0]]

heuristics_scores = heuristics(distance_matrix)
print(heuristics_scores)
```

This function incorporates a dynamic scaling factor, which could be tuned based on specific requirements. In this example, I've used a small constant scaling factor, but in a real application, you might adjust it based on the scale of your distances or performance metrics of your model.

The scoring matrix reflects edges with larger distances and/or those where both endpoints are far from the graph's center (in this case, node 0) as more undesirable.
