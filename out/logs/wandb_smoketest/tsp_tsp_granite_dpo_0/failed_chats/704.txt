This method is based on the idea that the total distance between two points, A to B, and B to C, should be at least as great as the direct distance between A and C (this is known as the triangle inequality). By violating this inequality, we're essentially suggesting that an edge might be a bad choice for including in an optimal solution.

```python
import numpy as np

def heuristics(distance_matrix):
    m, n = distance_matrix.shape
    heuristic_matrix = np.copy(distance_matrix)  # Start with the input distance matrix

    for i in range(m):
        for j in range(n):
            # Check the triangle inequality for every edge (i, j)
            if i!= j:
                # Check if j is closer to k (for some k > j) than it is to i
                for k in range(j+1, n):
                    if distance_matrix[i, k] < distance_matrix[j, k]:
                        # If yes, increment the heuristic value for edge (i, j)
                        heuristic_matrix[i, j] += 100  # Arbitrarily scale the heuristic value
                        heuristic_matrix[j, i] += 100  # Reflect the change in the transpose matrix

    return heuristic_matrix
```
