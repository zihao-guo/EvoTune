One of the popular heuristics for TSP and similar optimization problems is the Nearest Neighbor (NN) heuristic. It's simple and works well for small-to-medium size problems. This heuristic always picks the closest unvisited node at each step, which tends to produce better results, especially in terms of length, than the brute-force approach of visiting each node exactly once (which would have a score of 0).

Here's a Python function implementing the NN heuristic:

```python
def heuristics(distance_matrix):
    num_vertices = len(distance_matrix)
    # Initiate a distance matrix of the same shape with large values
    unvisited_distances = [[float('inf')] * num_vertices for _ in range(num_vertices)]

    # Set the distance to self as 0
    for i in range(num_vertices):
        unvisited_distances[i][i] = 0

    # NN heuristic: always pick the nearest unvisited vertex
    for _ in range(num_vertices):
        # Get the index of the nearest unvisited vertex
        min_dist_index = min(range(num_vertices), key=lambda i: unvisited_distances[0][i])

        # Mark this vertex as visited
        unvisited_distances[0][min_dist_index] = 0
        for j in range(num_vertices):
            # Update the distance if it's closer
            unvisited_distances[j][min_dist_index] = distance_matrix[j][min_dist_index]

    return unvisited_distances
```

This function initializes a matrix of unvisited distances and starts from the first vertex. At each iteration, it picks the vertex that's nearest to the current one and marks it as visited. It then updates the distances for all other vertices that were previously farther.

This method provides a rough estimate of the shortest possible route (or path with least weight, depending on the context), which is a heuristic that is easier to compute than the optimal solution. This can lead to a better performance or score compared to the previous heuristics().
