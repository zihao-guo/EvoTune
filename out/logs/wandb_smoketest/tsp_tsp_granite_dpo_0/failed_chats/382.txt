Let's design a heuristics function based on the 2-Opt move that aims to remove less critical edges first.

The 2-Opt heuristic works by iteratively cutting two edges in the tour, then reconnecting the resulting two shorter paths. It's assumed that edges with lower cost (i.e., smaller distances) contribute more to the overall cost, thus being more critical.

However, we need a way to rank edges in our distance matrix to decide which to remove first. We will achieve this by applying the ratio of the minimum to the second-minimum distance to each edge. Edges with a larger ratio are considered more critical.

Here's a Python function called `heuristics()` that applies this approach:

```python
import numpy as np

def heuristics(distance_matrix):
    # First, find the minimum and second-minimum distance for each row (i.e., each vertex)
    min_distances = np.min(distance_matrix, axis=1)
    sec_min_distances = np.sort(np.delete(np.min(distance_matrix, axis=1), 0))[::-1]

    # Create a new matrix where each value represents the 2-Opt ratio for each edge
    # An edge is represented as [u, v, cost] where u and v are the vertices and cost is the distance.
    # The 2-Opt ratio for an edge is (cost / sec_min_distance(u)) if sec_min_distance(u) > min_distances[v] else 0
    heuristic_matrix = np.where(
        (distance_matrix[:, None] >= distance_matrix[None, :]) & 
        (distance_matrix[:, None] < sec_min_distances[np.newaxis, :]),
        distance_matrix[:, None] / sec_min_distances[np.newaxis, :],
        0
    ).astype(float)

    return heuristic_matrix
```

This function returns a matrix where each element (i, j) represents the 2-Opt heuristic for the edge between vertices i and j. Edges with a larger 2-Opt ratio (i.e., larger cost divided by the second-minimum distance from vertex i) are more critical and thus should be removed first in order to minimize the total distance.
