This technique adds a penalty term to discourage edges that form a negative-weight cycle with previously included edges.

Here's a Python function that implements this idea:

```python
import numpy as np

def heuristics(distance_matrix):
    m, n = distance_matrix.shape
    
    # Initialize the heuristic matrix with zeros
    heuristic_matrix = np.zeros_like(distance_matrix)
    
    # Calculate the heuristic for each edge
    for i in range(m):
        for j in range(n):
            # Negative-weight cycle penalty
            if i!= j:
                heuristic_matrix[i, j] = distance_matrix[i, j]
                for k in range(m):
                    if k!= i and k!= j:
                        heuristic_matrix[i, j] += distance_matrix[i, k] + distance_matrix[k, j]
    
    # Convert negative values to a maximum possible positive value
    max_distance = np.max(heuristic_matrix[np.triu_indices(n)])
    heuristic_matrix[np.triu_indices(n)] = max_distance - heuristic_matrix[np.triu_indices(n)]
    
    return heuristic_matrix
```

This new heuristics() function adds a penalty for every possible negative-weight cycle formed with edges already included in the current tour. The negative values in the resulting heuristic matrix are then converted into a maximum possible positive value, ensuring the resulting scores are all non-negative.
