To create a more sophisticated heuristics() function, we'll incorporate a weighted shortest path approach. This approach considers not only the edge distance but also the number of intermediate nodes (or hops), which can often indicate a higher cost in real-world scenarios (e.g., traveling salesman problem). import numpy as np

def heuristics(distance_matrix):
    n = distance_matrix.shape[0]
    heuristics_matrix = np.zeros((n, n))

    for i in range(n):
        for j in range(n):
            # Shortest path distance via node i (using Floyd-Warshall algorithm)
            shortest_path_via_i = np.minimum.reduce([distance_matrix[i, k] + heuristics_matrix[k, j] for k in range(n) if k!= i])

            # If j can be reached via i, add the number of hops as a heuristic
            if shortest_path_via_i!= float('inf'):
                heuristics_matrix[i, j] = shortest_path_via_i + 1  # +1 for including the edge i-j
        else:
            # If i is a source node, set heuristic for all j to infinity
            heuristics_matrix[i, :] = float('inf')

    return heuristics_matrix

This function calculates the heuristics (estimated costs) based on the shortest path to each node via a given node, which incorporates additional hops. The '+1' adds the cost of including the edge in the path. This method should significantly improve the performance compared to the original functions, especially in problems with significant intermediate nodes or a topological structure that favors shorter paths.

Please note that the Floyd-Warshall algorithm is used for finding the shortest path via all intermediate nodes. This might not be the most efficient solution for sparse matrices (matrices with many zero entries), but it's appropriate for dense ones. For extremely large, sparse matrices, consider using Dijkstra's algorithm or its variant.
