This method, known as "weighted centrality", will take into account the average distance of each node (excluding the edge itself) to prioritize edges that connect nodes with high centrality.

Here is the Python function for the `heuristics()`:

```python
import numpy as np

def heuristics(distance_matrix):
    n = distance_matrix.shape[0]
    centrality = np.zeros((n, n))
    
    # Calculate average distances from each node to all other nodes
    for i in range(n):
        for j in range(n):
            if i!= j:
                centrality[i, j] = np.mean(distance_matrix[i, :j] + distance_matrix[j, :i])

    # Calculate weighted centrality
    weighted_centrality = np.zeros_like(distance_matrix)
    for i in range(n):
        for j in range(n):
            if distance_matrix[i, j] > 0:  # Assuming zero distance means no connection
                weighted_centrality[i, j] = centrality[i, j] / distance_matrix[i, j]

    # The heuristics matrix is the negative of weighted centrality
    heuristics_matrix = -weighted_centrality

    return heuristics_matrix

# Usage
distance_matrix = np.array([[0, 5, 8, 11],  # Example 4x4 matrix
                            [5, 0, 3, 12],
                            [8, 3, 0, 6],
                            [11, 12, 6, 0]])

print(heuristics(distance_matrix))
```

This function calculates the average distance of each node to all other nodes, then calculates a "weighted centrality" by dividing this average distance by the actual edge distance. This weighted centrality is used to create a heuristics matrix that penalizes more the edges that connect highly central nodes (i.e., nodes that are well-connected in the graph). This approach should provide a better overall score than the basic distance matrix.

As the `centrality` calculation involves two nested loops over a square matrix, its time complexity is O(n^2), the same as the original `distance_matrix` function. The space complexity is also O(n^2) due to the storage of `centrality` and `heuristics_matrix`.
