This heuristic improves solutions by iteratively swapping edges in the current solution. The 2-Opt algorithm aims to reduce the total distance by removing and replacing two edges with two others in the solution, under the condition that this operation results in a shorter total path.

However, instead of using TSP's total path, we'll work with the distance matrix to derive our heuristic values directly. The idea is to create a "cost-to-improve" value for each edge, which would encourage the function to optimize specific edges over others.

Here's a potential Python function named `heuristics()` that implements this approach:

```python
def heuristics(distance_matrix):
    num_vertices = distance_matrix.shape[0]
    
    # Initialize the heuristic matrix with positive values
    heuristic_matrix = np.full_like(distance_matrix, np.inf)
    
    # Calculate the cost-to-improve for each edge
    for i in range(num_vertices):
        for j in range(i+1, num_vertices):
            # Get the potential new total cost if edge (i, j) is removed
            new_cost = distance_matrix[i][j] + distance_matrix[i][-1] + distance_matrix[j][-1] - distance_matrix[i][-1] - distance_matrix[j][-1]
            
            # Update the heuristic if the new total cost is lower
            if new_cost < heuristic_matrix[i][j]:
                heuristic_matrix[i][j] = new_cost

    return heuristic_matrix
```
