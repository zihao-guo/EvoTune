This heuristic assigns a weight to each edge proportional to the distance between its endpoints, but it also takes into account the connectivity and the number of edges already used in the solution.

Here's a Python function implementing the Mihalis Yannakakis's heuristic:

```python
import numpy as np

def heuristics(distance_matrix):
    # Get the number of cities (vertices)
    n = distance_matrix.shape[0]
    
    # Initialize the heuristic matrix with the same size as the distance matrix
    heuristics_matrix = np.zeros((n, n))
    
    # Calculate the heuristic value for each edge
    for i in range(n):
        for j in range(n):
            if i!= j:  # Don't consider the diagonal (i.e., same vertex)
                # Proportional distance
                heuristics_matrix[i][j] = distance_matrix[i][j]
                # Connections and non-used edges factor
                num_connections = np.sum(distance_matrix[i, :] < distance_matrix[j, :])
                heuristics_matrix[i][j] *= (n - num_connections) / (n - 1)
    
    return heuristics_matrix
```

This function calculates the heuristic weight for each edge by considering the direct distance (proportional to the distance matrix) and the connectivity factor (number of shorter edges connected to the destination, normalized by (n-1) to avoid double-counting in the full graph).
