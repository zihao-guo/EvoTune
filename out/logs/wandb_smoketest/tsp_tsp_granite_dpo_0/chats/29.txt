The idea is to incorporate a heuristic score that considers both the direct distance and the distance to the nearest unvisited node for each edge. This approach aims to balance exploration and exploitation, making it more likely to select edges that lead to unexplored territory.

Here's an implementation of such an approach:

```python
import numpy as np

def heuristics(distance_matrix):
    m, n = distance_matrix.shape
    # Initialize the heuristic matrix with zeros
    heuristic_matrix = np.zeros((m, n))

    # Iterate through each node (i)
    for i in range(m):
        for j in range(n):
            if i!= j:
                # Calculate the shortest path from i to j
                min_path = np.min(distance_matrix[i, np.arange(m)] + distance_matrix[np.arange(m), j])
                # The heuristic for this edge is the sum of its distance and the distance to the nearest unvisited node
                heuristic_matrix[i, j] = distance_matrix[i, j] + min_path - distance_matrix[i, j]
            else:  # If i == j, there's no edge, so set the heuristic to 0
                heuristic_matrix[i, j] = 0

    return heuristic_matrix
```

This function calculates the total distance (direct distance) and the shortest path from each node to every other node. By adding these values, it provides a better estimate of the edge undesirability.
