```python
import numpy as np

def heuristics(distance_matrix):
    # Calculate the potential (or "heuristic score") for each edge
    num_vertices = distance_matrix.shape[0]
    potential = np.full((num_vertices, num_vertices), np.inf)
    
    for v1 in range(num_vertices):
        for v2 in range(v1, num_vertices):
            # Potential of a forward edge (v1 -> v2)
            potential[v1, v2] = min(potential[v1, v2], distance_matrix[v1, v2])
            # Potential of a backward edge (v2 -> v1)
            potential[v2, v1] = min(potential[v2, v1], potential[v1, v2] + distance_matrix[v1, v2])
    
    # Normalize the potential to create a heuristic score matrix
    potential = -np.minimum.accumulate(potential, axis=1)
    
    return potential
```

This function calculates an initial potential for every edge, considering both the edge itself and a "virtual" backward edge. The idea is that the potential of an edge (v1 -> v2) is the minimum of the direct edge cost and the sum of the direct edge cost and the backward potential.

After calculating the initial potentials, we use `np.minimum.accumulate` to create a heuristic score matrix. This function does a cumulative minimum, which effectively turns the potential matrix into a heuristic score matrix where each cell represents how undesirable it would be to include an edge in a solution. The negative sign is applied to convert this into a scoring system where lower values are better.

This method captures both the direct cost and the potential for "future cost reduction" when traversing a path, which could better represent an edge's undesirability compared to the simple distance matrix.
